# Требования к разработчику (Replit Agent)

## Общие принципы работы

### Эффективность
- **Работать эффективно**: Не тратить время на пустые действия, делать всё с первого раза правильно
- **Проверять по 10 раз**: Перед каждым изменением мысленно проверить 10 раз, что решение правильное и не создаст новых проблем
- **Думать наперёд**: Предвидеть возможные проблемы и решать их сразу, а не после жалоб пользователя

### Автоматизм
- **Автоматически использовать лучшие практики**: Не ждать указаний на каждую мелочь
- **Принимать микрорешения самостоятельно**: Для хорошего дизайна и UX делать очевидные решения без постоянных вопросов пользователю
- **Примеры микрорешений**:
  - Правильные отступы и padding (если всё выглядит прижатым - сразу исправить)
  - Размеры картинок на разных экранах (не делать слишком большими или маленькими)
  - Адекватные gap между элементами
  - Правильная типографика и читаемость

### Качество кода
- **Понятное ТЗ = качественная реализация**: Если дано чёткое ТЗ, выполнять его идеально
- **Не создавать новые проблемы при исправлении старых**: Консультироваться с архитектором при сложных изменениях
- **Тестировать на всех breakpoints**: Mobile, tablet, desktop - всё должно работать идеально

## Конкретные требования к текущему проекту

### Отступы и spacing
- **Большие отступы везде**: Ничто не должно быть прижато к краю экрана
- **Между блоками**: Достаточное расстояние, чтобы визуально разделять секции
- **Внутри блоков**: Комфортные отступы для чтения и восприятия
- **Padding от краёв**: На всех устройствах должны быть отступы от краёв экрана

### Адаптивность
- **Проверять на всех размерах**: <640px (mobile), 640-1023px (tablet), ≥1024px (desktop)
- **Плавные переходы**: Между breakpoints не должно быть резких скачков
- **Картинки**: Адекватные размеры на каждом устройстве (не слишком большие, не слишком маленькие)

### Дизайн
- **Цветовая палитра**: Soft green (softGreen объект с оттенками 50-600)
- **Соотношение сторон картинок**: 3:4 для всех product images
- **Консистентность**: Один стиль на всём лэндинге

## Что НЕ делать

❌ Не делать половинчатых решений, которые создают новые проблемы  
❌ Не спрашивать пользователя о каждой мелочи (отступы, размеры, очевидные UI решения)  
❌ Не оставлять код в сломанном состоянии  
❌ Не игнорировать рекомендации архитектора  

## Что ДЕЛАТЬ

✅ Думать комплексно о всей картине  
✅ Проверять решение со всех сторон перед реализацией  
✅ Консультироваться с архитектором при сложных изменениях  
✅ Делать сразу правильно, а не исправлять по 5 раз  
✅ Принимать очевидные дизайнерские решения самостоятельно
